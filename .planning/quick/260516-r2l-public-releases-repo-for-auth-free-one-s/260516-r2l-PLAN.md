<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI -> public repo | RELEASES_PAT allows write to public repo; must be scoped to public_repo only, not full repo |
| installer -> GitHub API | Unauthenticated requests to public API; rate-limited at 60 req/hr per IP, acceptable for install scripts |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-r2l-01 | Tampering | RELEASES_PAT secret | mitigate | Scope PAT to public_repo only; rotate annually; never log the value in CI output |
| T-r2l-02 | Spoofing | Public repo release assets | accept | Installer downloads whatever is tagged `latest` on the public repo; trust chain relies on GitHub account security. No code signing in scope for this phase. |
| T-r2l-03 | Denial of service | GitHub API rate limit | accept | 60 unauthenticated req/hr is ample for a one-shot install script; no caching layer needed |
| T-r2l-04 | Information disclosure | PAT in CI logs | mitigate | Use `${{ secrets.RELEASES_PAT }}` — GitHub Actions automatically masks secret values in logs |
</threat_model>
